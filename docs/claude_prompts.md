# Claude Code 別セッション用 プロンプト集

コンテキストの肥大化を防ぐため、以下の各ステップごとに別々の Claude Code セッションを立ち上げ（または直前に `/clear` を実行して）、順番に実行してください。

## 並列実行の可否について
- **Session 1** は全ての基盤となるため、最初に実行する必要があります。
- **Session 2, 3, 4** はそれぞれ独立した新規コンポーネントファイルの作成が主目的であるため、Session 1 完了後であれば**並列に実行可能**です。
- **Session 5** は全ての成果物を統合するため、最後に実行する必要があります。

以下のプロンプトを使用して、デジタル庁ガイドライン準拠のカラーパレットと大枠の `Dashboard` コンポーネントを作成します。

```text
テニスの王子様キャラクター年齢比較ダッシュボードを作っています。
デジタル庁のダッシュボードガイドラインに準拠した実装を行います。
以下の実装を行なってください。

1. `tailwind.config.ts` に、ガイドラインに準拠したHSLカラーパレット（青、落ち着いたグレー、白基調など）とフォント（Inter / Noto Sans JP）の設定を追加してください。
2. `src/app/globals.css` にベーススタイルとCSS変数を定義してください。
3. `src/components/Dashboard.tsx` を作成し、16:9比率・縦横6グリッドを意識したダッシュボードの全体レイアウトの枠組みを実装してください。
4. `Dashboard.tsx` 内部で、ユーザーの年齢を入力・保持するステート (`age: number | null`) と、学校・学年・年齢ポジションなどのフィルター用のステート枠組みだけを定義しておいてください（UIの詳細は次で行います）。

使用するデータは `src/data/characters.ts` に定義済みです。データのインポートが通るようにしておいてください。
```

---

## Session 2: フィルターUIとサマリーカード(KPI)の実装

以下のプロンプトを使用して、入出力機能とまとめ情報（KPI）の部分を作ります。

```text
テニスの王子様キャラクター年齢比較ダッシュボードの続きです。

1. `src/components/SummaryCards.tsx` を作成してください。
   - propsとして「フィルター条件」「ユーザーの年齢入力値」「キャラクターの配列」を受け取り、表示するデータを計算します。
   - 「同い年のキャラクター数」「年上のキャラクター数」「年下のキャラクター数」などのKPIを計算し、カード形式で表示してください。
2. `src/components/Dashboard.tsx` を更新し、上部または左部にフィルターUI（マルチセレクトなど）と年齢入力フィールドを実装してください。
3. `Dashboard.tsx` 内で `SummaryCards` を読み込み、レイアウトに沿って配置してください。

UIは、Tailwind CSSを活用しデジタル庁ガイドライン（視認性の高い配色、Zの法則による配置）に従った見やすいものにしてください。
```

---

## Session 3: グラフの実装 (recharts)

以下のプロンプトを使用して、データの視覚化を行います。

```text
テニスの王子様キャラクター年齢比較ダッシュボードの続きです。グラフ（Charts）を実装します。

1. `recharts` はインストール済みです。
2. `src/components/Charts.tsx` を新規作成してください。
3. 以下の2つのグラフを実装してください：
   - 「学校ごとのキャラクター数」を比較する棒グラフ（BarChart）
   - 「年齢分布（各年齢に何人いるか）」を示す棒グラフ（BarChart または ヒストグラム）
4. それぞれのグラフに対し、フィルターされたキャラクター配列をpropsとして渡し、動的に描画するようにしてください。
5. デジタル庁ガイドラインに従い、見やすく主張の強すぎない配色（設定済みのCSS変数などを活用）を用い、過度な装飾は避けてください。
6. `Dashboard.tsx` を更新し、`Charts.tsx` をレイアウト内の適切なグリッドに配置してください。
```

---

## Session 4: キャラクター詳細データ一覧表の実装

以下のプロンプトを使用して、表組み部分を作ります。

```text
テニスの王子様キャラクター年齢比較ダッシュボードの続きです。キャラクターの一覧表を実装します。

1. `src/components/CharacterTable.tsx` を新規作成してください。
2. フィルターされたキャラクター配列と、ユーザーの入力年齢をpropsとして受け取ります。
3. テーブルにはキャラクターの「名前, 学校, 学年, 年齢, 身長, 体重, 血液型, 利き腕, 家族」を表示してください。
4. **重要**: ユーザーが年齢を入力している場合、テーブルに「年齢差」という新しいカラムを動的に追加し、「+2（自分より2歳年上）」「同い年」「-1（自分より1歳年下）」などのように表示してください。
5. テーブルヘッダーの固定、ストライプ背景（行ごとの色分け）など、ガイドライン準拠の見やすい表デザインにしてください。
6. `Dashboard.tsx` に `CharacterTable.tsx` を配置してください。
```

---

## Session 5: ページルーティングと最終結合

以下のプロンプトを使用して、最終的な画面を組み上げます。

```text
テニスの王子様ダッシュボードの最終調整です。

1. `src/app/page.tsx` を更新し、作成した `Dashboard` コンポーネントをメインコンテンツとして読み込んでください。
2. ページ全体に、デジタル庁ガイドラインに準拠したヘッダー（サービス名など）と簡素なフッターを追加し、レイアウトを整えてください。
3. `npm run build` でエラーが出ないか、Typescriptの型エラーがないかチェックし、必要に応じてインポートや型定義を微調整してください。
4. 全体の余白や配置のバランス（16:9比率への適応性など）が崩れていないか確認し、必要なら Tailwind クラスを調整してください。
```

## 運用時の注意事項
- それぞれのプロンプトを投げる前に `claude` (または `claude-code`) を新しく立ち上げ直すか、`/clear` コマンドでセッションのリセットを行ってください。
- 前のセッションで作成したファイルをClaudeが正しく認識できるよう、新しいセッションの最初にファイルの存在確認 (`ls` 等) が行われる場合があります。
