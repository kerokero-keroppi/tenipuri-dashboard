# ダッシュボードガイドライン

デジタル庁「ダッシュボードデザイン 実践ガイドブック」に基づき、見やすく直感的な「提示型ダッシュボード」を構築するためのガイドラインです。

## 1. 原則 (Principles)
- **目的に則する**: ユーザーの判断や行動に直結するデータ（ユーザー年齢とキャラクター年齢の比較など）を最優先で表示する。
- **違いに気づける**: 平均やユーザーの入力値（年齢）とデータの差分がひと目で分かる視覚的強調（色や大きさ）を用いる。
- **適切な情報量**: 一画面に情報を詰め込まず、サマリーから詳細へと段階的にドリルダウンできるようにする。
- **比較対象を提供する**: 必ず基準値（今回の場合は「ユーザーの年齢」や「学校の平均」）を併記し、比較を行えるようにする。

## 2. レイアウト (Layout)
- **16:9の画面比率 (1920x1080想定)** を基準に、縦横6分割のグリッドレイアウトを使用する。
- **Zの法則 / Fの法則**:
  - 左上に最も重要な全体サマリー（ユーザーとキャラの年齢差ハイライトなど）
  - 右下に向かって詳細なデータ（各キャラクターごとの一覧や表など）
- **フィルター配置**: 画面上部または左部に配置し、フィルターのスコープが直感的に伝わるようにする。

## 3. コンポーネントとグラフの選択 (Chart Selection)
- **折れ線グラフ**: 時間変化（今回は該当しない可能性が高い）
- **棒グラフ**: 学校別、学年別のキャラクター数・平均年齢・平均身長・平均体重の比較
- **表組み**: キャラクターの詳細データ一覧表示（身長、体重、血液型などの一覧比較）
- **指標（KPIカード）**: 単一の数値（例：同い年のキャラクター数、ユーザーより年上のキャラクター数など）

## 4. 配色とタイポグラフィ (Colors & Typography)
- デジタル庁の公式アセット（powerbi-theme-json）で提供されているテーマカラーセットを元に配色する。
- テーマカラーの構成（計8色）の原則:
  - **テーマ色階調（1〜5色目）**: メインテーマカラーの濃から淡までの5段階グラデーション（例: 青テーマなら `#0017C1` 〜 `#E8F1FE`）。
  - **ハイライト・セマンティック色（6〜7色目）**: 強調（例: 青 `#3460FB`）や、異常・注意・警告を意味する色（例: 赤 `#FE3939`）。
  - **背景色（8色目）**: ダッシュボード全体の背景用のうすい色（例: `#F8F8FB`）。
- 汎用的な原色は避け、上記の配色ルールに基づいたカラーパレットを適用し、システム全体で「正常」「異常・注目」がひと目で分かるようにする。
- フォントは視認性に優れたモダンタイポグラフィ（例: Inter, Roboto あるいは Noto Sans JP）を利用する。

## 5. UI/UX (インタラクション)
- データホバー時のツールチップ表示、ソート機能、絞り込み機能を備え、直感的にデータを「探索」できる余地を残す。
- フィルターの切り替え時に過剰なリロードを挟まないスムーズな実装。
